Navigation Path Bar

HTML Structure

The path bar is implemented as a DIV container with three DIV child elements: A content container, a left arrow button, and a right arrow button. The content container contains a list of parent node links, from the home page to the current document parent. The buttons become visible if the width of the parent node link list is wider than the viewport window.

<div class="path-bar">
    <div class="content">parent node link list</div>
    <div class="scroll left"></div>
    <div class="scroll right"></div>
</div>

CSS Code

The path bar is a modified DIV, a block element that spans the entire width. The height is set explicitly so that we have some value to work with for the child elements inside. Position is relative so that we can position child elements absolute:

div.path-bar { 
	border : 1px darkorange ;
	border-style : none none solid none ;
	height : 30px ;
	position : relative ;
	overflow : hidden ;
	user-select : none ;
	}

Overflow is hidden to prevent a horizontal scroll bar – which would look rather awkwardly here. User-select none prevents the content to be selected if the user clicks around in the text.

The path-bar container holds inline-block DIV elements with absolute position. This is necessary to control their horizontal position:

div.path-bar div { 
	display : inline-block ;
	position : absolute ;
	width : auto ; 
	vertical-align : middle ;
	border : 1px none silver ;
}

The content DIV holds the link list in a single line. Over

	}

	}

	}
